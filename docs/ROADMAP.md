# 노션 기반 견적서 관리 시스템 개발 로드맵

노션을 데이터베이스로 활용하여 견적서를 관리하고, 클라이언트가 웹에서 조회 및 PDF 다운로드할 수 있는 시스템

## 개요

노션 기반 견적서 관리 시스템은 프리랜서 및 소규모 기업을 위한 견적서 발행/관리 솔루션으로 다음 기능을 제공합니다:

- **노션 데이터베이스 연동**: Notion API를 통해 견적서 데이터를 실시간 조회
- **견적서 웹 조회**: 고유 URL을 통한 견적서 내용 표시 및 공유
- **PDF 다운로드**: @react-pdf/renderer 기반 견적서 PDF 변환 및 다운로드
- **관리자 대시보드**: 발행한 견적서 목록 관리 및 상태 추적 (예정)
- **자동화**: 이메일 발송, 만료 알림, 클라이언트 응답 트래킹 (예정)

## 기술 스택

- Next.js 15.5.3 (App Router), React 19, TypeScript 5.x
- TailwindCSS v4, shadcn/ui, Lucide React
- @notionhq/client (Notion API v1)
- @react-pdf/renderer (PDF 생성)
- Vercel (배포)

## 개발 워크플로우

1. **작업 계획**

   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입

2. **작업 생성**

   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - `/tasks` 디렉토리에 새 작업 파일 생성
   - 명명 형식: `XXX-description.md` (예: `001-setup.md`)
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - API/비즈니스 로직 작업 시 "## 테스트 체크리스트" 섹션 필수 포함 (Playwright MCP 테스트 시나리오 작성)
   - 예시를 위해 `/tasks` 디렉토리의 마지막 완료된 작업 참조

3. **작업 구현**

   - 작업 파일의 명세서를 따름
   - 기능과 기능성 구현
   - API 연동 및 비즈니스 로직 구현 시 Playwright MCP로 테스트 수행 필수
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트
   - 각 단계 완료 후 중단하고 추가 지시를 기다림

4. **로드맵 업데이트**

   - 로드맵에서 완료된 작업을 완료 상태로 표시

- **Task 003: 견적서 조회 페이지 구현** ✅ - 완료

  - ✅ 파일명 자동 생성 (invoice_{견적번호}.pdf)

- **Task 005: 에러 처리 및 유효성 검증** ✅ - 완료

### Phase 3: 자동화

> 이메일 자동 발송, 견적서 만료 알림, 클라이언트 응답 트래킹 기능 구현

- **Task 012: 이메일 발송 시스템 구축**
  - 이메일 서비스 연동 (Resend 또는 SendGrid SDK 설정)
  - 견적서 공유 이메일 템플릿 작성 (HTML 이메일)
  - 관리자 대시보드에서 이메일 발송 UI 구현
  - 이메일 발송 API Route (`app/api/send-email/route.ts`)
  - 발송 이력 관리 (노션 프로퍼티에 발송 일시 기록)
  - Playwright MCP를 활용한 이메일 발송 플로우 테스트

- **Task 013: 견적서 만료 알림 시스템**
  - 유효기간 만료 임박 견적서 감지 로직 구현
  - Vercel Cron Jobs 또는 외부 스케줄러 연동
  - 만료 알림 이메일 자동 발송 (D-7, D-3, D-day)
  - 관리자 대시보드에 만료 임박 알림 배너 표시
  - 만료된 견적서 자동 상태 변경 (expired)
  - Playwright MCP를 활용한 만료 알림 로직 테스트

- **Task 014: 클라이언트 응답 트래킹**
  - 견적서 조회 이벤트 기록 (조회 시간, IP, User-Agent)
  - 노션 데이터베이스에 조회 기록 저장 (별도 DB 또는 프로퍼티)
  - PDF 다운로드 이벤트 트래킹
  - 관리자 대시보드에 트래킹 데이터 표시 (최근 조회, 다운로드 횟수)
  - 클라이언트 활동 타임라인 UI
  - Playwright MCP를 활용한 트래킹 데이터 기록 및 표시 테스트

- **Task 015: Phase 3 통합 테스트**
  - 이메일 발송 -> 클라이언트 견적서 조회 -> 트래킹 기록 전체 플로우 테스트
  - 만료 알림 스케줄링 검증
  - 에러 핸들링 (이메일 발송 실패, API 한도 초과) 테스트
